#### 六边形碰撞检测算法
```javascript
function isPointInHexagon(px, py, centerX, centerY, size) {
    const dx = Math.abs(px - centerX);
    const dy = Math.abs(py - centerY);
    const size3_2 = size * Math.sqrt(3) / 2;

    // 快速排除：超出矩形范围
    if (dx > size || dy > size3_2) return false;

    // 六边形内部判断
    return (size3_2 * dx + size * dy / 2) <= size * size3_2;
}
```

#### 网格与矩形相交检测（AABB）
```javascript
function isGridInRectangle(grid, rectLeft, rectTop, rectRight, rectBottom) {
    // 使用六边形外接矩形进行AABB检测
    const hexLeft = centerX - r;
    const hexRight = centerX + r;
    const hexTop = centerY - r * Math.sqrt(3) / 2;
    const hexBottom = centerY + r * Math.sqrt(3) / 2;

    return !(hexRight < rectLeft || hexLeft > rectRight ||
             hexBottom < rectTop || hexTop > rectBottom);
}
```

- 边框：`rgba(74, 222, 128, 0.8)` 绿色虚线，线宽 2px
- 虚线样式：`[5, 5]`

### 3.6 网格向量化坐标系统

#### 状态变量
```javascript
coordEnabled: boolean         // 坐标是否显示
gridCoordinates: Array<{      // 网格坐标数据
    gridId: string,           // 网格唯一ID (row_col)
    row: number,              // 行索引
    col: number,              // 列索引
    x: number,                // x 坐标（原点在右下角）
    y: number,                // y 坐标（原点在右下角）
    centerXNatural: number,   // 自然坐标系下的中心 X
    centerYNatural: number,   // 自然坐标系下的中心 Y
    hexSizeNatural: number,   // 自然坐标系下的六边形大小
    neighbors: Array<string>  // 邻居网格的 gridId 数组
}>
```

#### 坐标系设计
```
                    y 轴
                    ↑
                    |
                    |
         (0, W) ┌───┼───┐ (H, W)
                 │   │   │
                 │   │   │
         (0, 0) └───┼───┘ (H, 0)
                    |
         <──────────┼────────── x 轴
                    |
```

- **坐标系类型**：二维笛卡尔坐标系，第一象限
- **原点位置**：图片右下角顶点 (0, 0)
- **x 轴方向**：向左为正方向
- **y 轴方向**：向上为正方向
- **坐标单位**：像素（基于图片自然尺寸）
- **坐标定义**：网格中心的坐标即为该网格的坐标

#### 坐标计算公式
```javascript
// 图片自然尺寸
naturalWidth = previewImage.naturalWidth
naturalHeight = previewImage.naturalHeight

// 网格中心在自然坐标系中的位置（左上角为原点）
centerXNatural = (col * xDist + (row % 2 === 1 ? xDist / 2 : 0)) / imageScale
centerYNatural = (row * yDist) / imageScale

// 转换为右下角为原点的坐标系
x = naturalWidth - centerXNatural
y = naturalHeight - centerYNatural
```

**坐标系统转换关系：**

```javascript

// 从向量化坐标系转换回自然坐标系
centerXNatural = naturalWidth - x;
centerYNatural = naturalHeight - y;
```

**使用说明：**
1. `x` 和 `y` 是主要的向量化坐标，用于定位网格
2. `row` 和 `col` 可用于网格的逻辑定位
3. `centerXNatural` 和 `centerYNatural` 可用于在原始图片上进行像素级操作
4. `hexSizeNatural` 可用于计算六边形的实际覆盖范围
5. `neighbors` 可用于快速访问相邻网格，实现网格间的关联分析
6. `color` 和 `colorTag` 可用于识别网格的类型（森林密集区、水坑等）

### 3.6.1 网格邻居计算

#### 邻居定义
网格 A 相邻的网格即为网格 A 的网格邻居或网格 A 的邻居网格。六边形网格最多有 6 个邻居。

#### 实现方式
- 分两步生成网格数据：
  1. 第一步：遍历所有网格，生成网格对象并存入 `gridMap`（Map&lt;gridId, gridData&gt;）
  2. 第二步：再次遍历所有网格，为每个网格计算邻居
- 使用 `gridMap` 快速查找邻居网格是否存在

#### 六边形邻居偏移表

本项目使用的是**偏移坐标系统（odd-r）**，奇数行向右偏移。

**偶数行（row % 2 === 0）的邻居偏移：**
| 位置 | 行偏移 (dr) | 列偏移 (dc) |
|------|-------------|-------------|
| 左上 | -1          | 0           |
| 右上 | -1          | -1          |
| 左   | 0           | -1          |
| 右   | 0           | 1           |
| 左下 | 1           | 0           |
| 右下 | 1           | -1          |

**奇数行（row % 2 === 1）的邻居偏移：**
| 位置 | 行偏移 (dr) | 列偏移 (dc) |
|------|-------------|-------------|
| 左上 | -1          | 1           |
| 右上 | -1          | 0           |
| 左   | 0           | -1          |
| 右   | 0           | 1           |
| 左下 | 1           | 1           |
| 右下 | 1           | 0           |

#### 邻居计算代码逻辑
```javascript
// 确定当前行的邻居偏移表
let neighborOffsets;
if (row % 2 === 0) {
    neighborOffsets = [
        [-1, 0],    // 左上
        [-1, -1],   // 右上
        [0, -1],    // 左
        [0, 1],     // 右
        [1, 0],     // 左下
        [1, -1]     // 右下
    ];
} else {
    neighborOffsets = [
        [-1, 1],    // 左上
        [-1, 0],    // 右上
        [0, -1],    // 左
        [0, 1],     // 右
        [1, 1],     // 左下
        [1, 0]      // 右下
    ];
}

// 查找邻居
for (const [dr, dc] of neighborOffsets) {
    const neighborRow = row + dr;
    const neighborCol = col + dc;
    const neighborId = `${neighborRow}_${neighborCol}`;
    if (gridMap.has(neighborId)) {
        neighbors.push(neighborId);
    }
}
```

### 3.6.2 SVG矢量图导出

#### 功能说明
- 支持将网格图层导出为SVG矢量图
- 导出包含所有网格（有填充和无填充的都导出）
- 导出使用图片自然尺寸作为画布尺寸
- 导出文件名为 `grid-layer.svg`
- 网格边框固定为黑色
- 保持网格的填充属性（有填充的显示填充，无填充的不填充）

#### SVG结构
```xml
<?xml version="1.0" encoding="UTF-8"?>
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 W H" width="W" height="H">
  <!-- Hex Grid Layer -->
  <!-- 有填充颜色的网格 -->
  <polygon points="x1,y1 x2,y2 ..." fill="color" fill-opacity="0.8"
           stroke="#000000" stroke-opacity="1" stroke-width="1"
           data-grid-id="row_col"/>
  <!-- 无填充颜色的网格 -->
  <polygon points="x1,y1 x2,y2 ..." fill="none"
           stroke="#000000" stroke-opacity="1" stroke-width="1"
           data-grid-id="row_col"/>
</svg>
```

#### 六边形SVG属性
| 属性名 | 值 | 说明 |
|--------|-----|------|
| `points` | 字符串 | 六边形六个顶点的坐标，格式为 "x1,y1 x2,y2 ..." |
| `fill` | 颜色值 \| "none" | 网格填充颜色，无填充时为 "none" |
| `fill-opacity` | 0.8 | 填充不透明度（仅在有填充时设置） |
| `stroke` | #000000 | 边框颜色（固定黑色） |
| `stroke-opacity` | 1 | 边框不透明度 |
| `stroke-width` | 1 | 边框宽度 |
| `data-grid-id` | 字符串 | 网格ID（自定义属性） |

#### 实现方式
- 使用图片自然尺寸（`naturalWidth` × `naturalHeight`）作为SVG画布尺寸
- 使用自然尺寸的六边形半径（`hexSize`）计算网格位置
- 所有网格都生成 `<polygon>` 元素
- 有填充颜色的网格：`fill` 设置为颜色值，`fill-opacity` 为 0.8
- 无填充颜色的网格：`fill` 设置为 "none"
- 所有网格的边框都固定为黑色（`stroke="#000000"`），不透明度为 1
- 六边形顶点从 -30度开始计算
- 在图片右下角添加颜色图例（legend）
- 使用 Blob URL 方式下载SVG文件

#### 图例（Legend）设计

**图例配置**
| 参数 | 计算方式/值 | 说明 |
|------|-------------|------|
| 位置 | 右下角 | 距离右边和下边各 `1.5 × 字体大小` |
| 背景 | `rgba(0,0,0,0.5)` | 半透明黑色背景 |
| 圆角 | 5px | 背景圆角半径 |

### 3.7 网格悬停高亮模块

#### 状态变量
```javascript
hoveredGridId: string | null  // 当前鼠标悬停的网格ID
```

#### 实现方式
- 在 `mousemove` 事件中检测鼠标当前悬停的网格
- 使用六边形碰撞检测算法判断鼠标是否在网格内
- 当悬停网格变化时，更新 `hoveredGridId` 并重绘网格
- 在 `mouseleave` 事件中清除 `hoveredGridId`

#### 邻居网格高亮
- 绘制网格前，先获取悬停网格的邻居集合
- 遍历所有网格时，判断当前网格是否在邻居集合中
- 将 `isNeighbor` 参数传递给 `drawHexagon` 函数
- 邻居网格的边框加粗到 2px

#### 悬停网格ID显示
- 在所有网格绘制完成后，额外绘制悬停网格的ID
- 使用粗体字体，字号比坐标标签稍大
- 添加黑色文字阴影提高可读性
- 文字颜色为白色，居中显示在网格中心

#### 绘制流程
```
1. 检测悬停网格 → 更新 hoveredGridId
2. 获取悬停网格的邻居列表 → 存入 neighborGridIds Set
3. 遍历所有网格：
   - 判断是否是选中网格 (isHighlighted)
   - 判断是否有自定义颜色 (hasCustomColor)
   - 判断是否是邻居网格 (isNeighbor)
   - 调用 drawHexagon 绘制
4. 绘制悬停网格的ID（如果有）
```

#### 事件处理
- `mousemove`：检测悬停网格 + 处理框选
- `mouseleave`：清除悬停状态

### 3.9 网格颜色设置模块

#### 状态变量
```javascript
currentGridColor: string   // 当前选择的颜色，默认值: '#4ade80'
defaultGridColor: string   // 默认网格颜色，默认值: '#4ade80'
gridColorMap: Map<string, string>  // 网格ID到颜色的映射
```

#### 实现方式
- 使用 HTML5 `<input type="color">` 颜色选择器控件
- 默认颜色为绿色 `#4ade80`
- 每个网格可以有独立的颜色，使用 `gridColorMap` 保存
- 颜色变化逻辑：
  - **有选中网格时**：只改变选中网格的颜色，其他网格保持不变
  - **无选中网格时**：设置为默认颜色，新绘制的网格使用此颜色
- 网格绘制逻辑：
  - **有自定义颜色的网格**：一直保持填充状态，不管是否选中
  - **选中的网格**：在填充基础上再加粗边框
  - **无自定义颜色且未选中的网格**：只显示默认颜色边框
- 重置时清空颜色映射并恢复默认颜色

#### 颜色转换函数
```javascript
function hexToRgba(hex, alpha) {
    const r = parseInt(hex.slice(1, 3), 16);
    const g = parseInt(hex.slice(3, 5), 16);
    const b = parseInt(hex.slice(5, 7), 16);
    return `rgba(${r}, ${g}, ${b}, ${alpha})`;
}
```

#### 颜色选择器样式
- 尺寸：36px × 36px
- 边框：2px 实线，悬停时变为主题色
- 圆角：4px
- 内置颜色块无边框
