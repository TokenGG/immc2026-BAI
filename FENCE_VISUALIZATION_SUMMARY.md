# 围栏可视化问题诊断和修复

## 问题描述

可视化脚本在热力图上显示所有允许部署围栏的边缘网格都有围栏标记，而不仅仅是实际部署的围栏。

## 修复方案

### 1. 改进的可视化脚本

**文件**：`hexdynamic/visualize_output.py`

**改进**：
- 在 `_draw_resources()` 函数中添加了调试信息
- 清晰的注释说明围栏只在实际部署的位置显示
- 打印围栏部署的统计信息

**调试输出示例**：
```
[DEBUG] 实际部署的围栏边数: 5
[DEBUG] 边缘网格数: 24
[DEBUG] 显示围栏标记的网格数: 8
```

### 2. 诊断脚本

**文件**：`diagnose_fence_deployment.py`

**功能**：
- 检查约束条件中的围栏长度
- 分析输出JSON中的围栏部署情况
- 计算边缘网格和围栏覆盖率
- 提供诊断结论和建议

**使用方法**：
```bash
python diagnose_fence_deployment.py output.json
python diagnose_fence_deployment.py output.json --input input.json
```

## 问题排查步骤

### 步骤1：运行诊断脚本

```bash
python diagnose_fence_deployment.py output.json --input input.json
```

### 步骤2：检查诊断结论

根据诊断脚本的输出，判断问题类型：

#### 情况A：没有部署任何围栏
```
❌ 问题：没有部署任何围栏
```

**解决方案**：
1. 检查输入JSON中的 `total_fence_length` 是否 > 0
2. 增加 `total_fence_length` 的值
3. 检查地图是否有边界网格

#### 情况B：所有边缘网格都是围栏端点
```
⚠️  警告：所有边缘网格都是围栏端点
```

**解决方案**：
1. 这可能是正常的（围栏覆盖整个边界）
2. 或者可视化脚本有问题
3. 运行可视化脚本查看热力图

#### 情况C：部分边缘网格有围栏（正常）
```
✅ 正常：部分边缘网格有围栏
围栏覆盖率: 33.3%
```

**说明**：
- 这是正常的部署情况
- 围栏只在必要的位置部署
- 可以运行可视化脚本查看热力图

### 步骤3：运行可视化脚本

```bash
python hexdynamic/visualize_output.py output.json --input input.json
```

查看生成的热力图，检查围栏是否正确显示。

### 步骤4：检查热力图

- **正确的显示**：围栏标记只出现在实际部署的位置
- **错误的显示**：围栏标记遍布所有边缘网格

## 围栏部署的正确行为

### 围栏的作用

- 在保护区边界部署围栏
- 增加边界网格的保护效果
- 防止非法入侵

### 围栏的显示

- **标记**：五边形（pentagon）
- **颜色**：红色（#c0392b）
- **位置**：在网格中心上方
- **数量**：只在实际部署的围栏端点显示

### 围栏的优化

- 优化器根据风险分布决定部署位置
- 优先在高风险区域的边界部署
- 受约束条件限制（total_fence_length）

## 常见问题

### Q1：为什么所有边缘网格都显示围栏？

**A**：可能的原因：
1. 输出JSON中的 `fence_edges` 包含所有边缘网格
2. 可视化脚本的逻辑有问题
3. 优化器部署了大量围栏

**解决方案**：
- 运行诊断脚本检查实际部署的围栏数
- 查看输出JSON中的 `fence_edges` 字段
- 检查可视化脚本的逻辑

### Q2：为什么没有显示任何围栏？

**A**：可能的原因：
1. 优化器没有部署任何围栏
2. 约束条件中 `total_fence_length = 0`
3. 可视化脚本的逻辑有问题

**解决方案**：
- 运行诊断脚本检查围栏部署情况
- 增加 `total_fence_length` 的值
- 检查可视化脚本的逻辑

### Q3：围栏显示位置不对？

**A**：可能的原因：
1. 六边形坐标计算有误
2. 网格中心计算有误
3. 边缘网格计算有误

**解决方案**：
- 检查 `grid_center()` 函数
- 检查 `_edge_grid_ids()` 函数
- 查看六边形的几何参数
