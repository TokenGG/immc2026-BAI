# 资源部署总结功能

## 功能说明

在DSSA优化迭代结束后，系统会自动输出详细的资源部署总结，帮助用户快速了解优化结果。

## 输出内容

### 1. 资源部署详情

总结包含以下资源的部署情况：

#### 📷 摄像头 (Cameras)
- 部署数量：实际部署数 / 总可用数
- 部署位置：部署的网格数量
- 主要部署：显示部署最多的前5个网格及数量

#### 🚁 无人机 (Drones)
- 部署数量：实际部署数 / 总可用数
- 部署位置：部署的网格数量
- 部署网格：列出前10个部署网格的ID

#### ⛺ 营地 (Camps)
- 部署数量：实际部署数 / 总可用数
- 部署网格：列出所有营地所在的网格ID

#### 👮 巡逻人员 (Rangers)
- 部署数量：实际部署数 / 总可用数
- 部署位置：部署的网格数量
- 主要部署：显示人员最多的前10个网格及人数

#### 🚧 围栏 (Fences)
- 部署段数：实际部署的围栏段数
- 示例边：显示前5条围栏边的网格连接

### 2. 部署统计

显示各类资源的利用率：
- 摄像头利用率
- 无人机利用率
- 营地利用率
- 巡逻人员利用率

### 3. 最终指标

优化完成后显示：
- 最佳适应度 (Best Fitness)
- 总保护收益 (Total Protection Benefit)
- 平均保护收益 (Average Protection Benefit)
- 总风险 (Total Risk)
- 网格总数 (Total Grids)

## 使用方法

运行优化pipeline时，总结会自动显示：

```bash
python hexdynamic/protection_pipeline.py input.json output.json
```

或使用向量化版本：

```bash
python hexdynamic/protection_pipeline.py input.json output.json --vectorized
```

## 实现细节

修改文件：`hexdynamic/protection_pipeline.py`

在`run_pipeline`函数中，优化完成后（`optimizer.optimize()`之后）添加了资源部署总结的输出代码。

总结信息包括：
1. 统计各类资源的实际部署数量
2. 计算资源利用率
3. 显示主要部署位置
4. 格式化输出，使用emoji图标增强可读性

## 优势

1. **直观清晰**：一目了然地看到所有资源的部署情况
2. **便于分析**：快速识别资源利用率和部署模式
3. **易于调试**：帮助发现部署异常或约束违反
4. **用户友好**：使用emoji和格式化输出，提升可读性

## 注意事项

- 总结在优化完成后、写入JSON文件前显示
- 如果某类资源未部署，相关信息仍会显示（数量为0）
- 利用率计算使用实际部署数除以约束中的总可用数
- 主要部署位置按数量降序排列，最多显示前5-10个
