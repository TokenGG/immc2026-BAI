# 为什么资源利用率为0%？深度解析

## 问题描述

即使资源约束不为0，且有足够空间部署，某些资源的利用率仍然为0%。

```
📊 部署统计:
   摄像头利用率: 0.0%    ← 为什么？约束是4个，有空间部署
   无人机利用率: 100.0%
   营地利用率: 0.0%      ← 为什么？约束是2个，有空间部署
   巡逻人员利用率: 100.0%
```

## 核心原因：边际收益递减

这不是bug，而是优化器的正确行为！原因是：

### 1. 适应度函数的数学原理

DSSA优化器的目标是最大化适应度函数：

```
fitness = total_benefit / total_risk
```

其中：

```
total_benefit = Σ(每个网格的保护收益)
每个网格的保护收益 = risk * (1 - exp(-E_i))
E_i = wp * patrol_cov + wd * drone_cov + wc * camera_cov + wf * fence_prot
```

### 2. 边际收益递减效应

关键在于 `1 - exp(-E_i)` 这个函数：

| E_i值 | 保护收益率 | 边际增益 |
|-------|-----------|---------|
| 0.0   | 0.00%     | -       |
| 0.2   | 18.13%    | 18.13%  |
| 0.4   | 32.97%    | 14.84%  |
| 0.6   | 45.12%    | 12.15%  |
| 0.8   | 55.07%    | 9.95%   |
| 1.0   | 63.21%    | 8.14%   |

**观察**：随着E_i增加，每增加0.2带来的边际收益在递减！

### 3. 实际案例分析

假设当前配置：
- wp = 0.3 (巡逻权重)
- wd = 0.3 (无人机权重)
- wc = 0.2 (摄像头权重)
- wf = 0.2 (围栏权重)

**场景1：只部署无人机**
```
无人机覆盖率 = 0.8
E_i = 0.3 * 0.8 = 0.24
保护收益率 = 1 - exp(-0.24) = 21.3%
```

**场景2：无人机 + 摄像头**
```
无人机覆盖率 = 0.8
摄像头覆盖率 = 0.6
E_i = 0.3 * 0.8 + 0.2 * 0.6 = 0.36
保护收益率 = 1 - exp(-0.36) = 30.2%
摄像头额外贡献 = 30.2% - 21.3% = 8.9%  ← 很小！
```

**场景3：无人机 + 巡逻**
```
无人机覆盖率 = 0.8
巡逻覆盖率 = 0.7
E_i = 0.3 * 0.8 + 0.3 * 0.7 = 0.45
保护收益率 = 1 - exp(-0.45) = 36.2%
巡逻额外贡献 = 36.2% - 21.3% = 14.9%  ← 比摄像头大！
```

**结论**：巡逻的贡献(14.9%)是摄像头(8.9%)的1.7倍，所以优化器选择部署巡逻而不是摄像头！

## 为什么会这样？

### 原因1：权重不平衡

```json
"coverage_params": {
    "wp": 0.3,  // 巡逻权重高
    "wd": 0.3,  // 无人机权重高
    "wc": 0.2,  // 摄像头权重低 ← 问题所在
    "wf": 0.2   // 围栏权重低
}
```

权重低的资源对E_i的贡献小，边际收益也小。

### 原因2：已有资源提供了足够保护

如果无人机和巡逻已经提供了E_i = 0.45的保护（保护收益率36.2%），再增加摄像头：

```
E_i = 0.45 + 0.12 = 0.57
保护收益率 = 1 - exp(-0.57) = 43.4%
摄像头额外贡献 = 43.4% - 36.2% = 7.2%  ← 更小了！
```

由于边际收益递减，摄像头的贡献变得更小。

### 原因3：优化器的理性选择

优化器会计算：
- 部署摄像头：增加7.2%的保护收益
- 不部署摄像头：节省资源，保持36.2%的保护收益

如果7.2%的增益不足以抵消部署成本（在适应度函数中的隐含成本），优化器会选择不部署。

## 营地的特殊情况

营地利用率为0%还有一个额外原因：

**营地本身不直接提供保护收益！**

在当前实现中：
- 营地只是巡逻人员的部署点
- 巡逻人员可以独立部署（不依赖营地）
- 所以优化器倾向于直接部署巡逻人员，而不是先建营地

这是设计决策，不是bug。如果需要强制使用营地，需要修改约束逻辑。

## 解决方案

### 方案1：平衡权重配置（推荐）

如果希望所有资源都被使用：

```json
"coverage_params": {
    "wp": 0.25,  // 平衡
    "wd": 0.25,  // 平衡
    "wc": 0.25,  // 平衡
    "wf": 0.25   // 平衡
}
```

这样每种资源的贡献更加平衡。

### 方案2：增加未使用资源的权重

如果摄像头利用率为0%：

```json
"coverage_params": {
    "wp": 0.25,  // 减少
    "wd": 0.25,  // 减少
    "wc": 0.35,  // 增加 ← 从0.2增加到0.35
    "wf": 0.15   // 减少
}
```

### 方案3：增加覆盖半径

增加摄像头的覆盖半径，提高其覆盖率：

```json
"coverage_params": {
    "camera_radius": 4.0  // 从2.0增加到4.0
}
```

覆盖率越高，对E_i的贡献越大，边际收益也越大。

### 方案4：减少其他资源数量

减少无人机数量，为摄像头腾出"收益空间"：

```json
"constraints": {
    "total_drones": 1,    // 从2减少到1
    "total_cameras": 4    // 保持不变
}
```

这样无人机提供的保护减少，摄像头的边际收益会增加。

### 方案5：接受优化结果（推荐）

如果其他资源已经提供了足够保护，不部署某些资源是合理的：

- 这是优化器的正确行为
- 说明资源配置已经很高效
- 不需要强制使用所有资源

## 数学证明

让我们用数学证明为什么会出现这种情况。

### 定理：边际收益递减

对于函数 `f(x) = 1 - exp(-x)`：

```
f'(x) = exp(-x)
f''(x) = -exp(-x) < 0  ← 二阶导数为负
```

二阶导数为负意味着函数是凹函数，具有边际收益递减特性。

### 推论：资源竞争

假设有两种资源A和B，权重分别为w_A和w_B，覆盖率分别为c_A和c_B。

如果 `w_A * c_A > w_B * c_B`，那么：

```
部署A的边际收益 > 部署B的边际收益
```

因此优化器会优先部署A而不是B。

### 实例验证

```
巡逻：w_p = 0.3, c_p = 0.7 → 贡献 = 0.21
摄像头：w_c = 0.2, c_c = 0.6 → 贡献 = 0.12

0.21 > 0.12 → 优化器选择巡逻
```

## 常见误解

### 误解1："有空间就应该部署"

**错误**：有空间不等于应该部署。优化器追求的是最大化保护收益，而不是最大化资源使用率。

**正确**：只有当部署某资源能显著增加保护收益时，才应该部署。

### 误解2："利用率为0%是bug"

**错误**：利用率为0%不一定是bug。

**正确**：如果其他资源已经提供了足够保护，不部署某些资源是优化器的理性选择。

### 误解3："所有资源都应该被使用"

**错误**：不是所有配置的资源都必须被使用。

**正确**：资源配置是上限，优化器会根据实际情况选择最优组合。

## 实际建议

### 对于研究人员

如果你在做实验，想要测试所有资源的效果：
- 使用平衡权重配置（0.25, 0.25, 0.25, 0.25）
- 或者分别测试每种资源的单独效果

### 对于实际应用

如果你在实际部署保护资源：
- 接受优化器的结果
- 利用率为0%说明该资源在当前配置下不是最优选择
- 可以考虑减少该资源的预算，增加其他资源

### 对于参数调优

如果你想找到最佳配置：
1. 先用默认配置运行
2. 观察哪些资源利用率低
3. 分析原因（权重？覆盖半径？）
4. 调整参数
5. 重新运行并比较结果
6. 选择总保护收益最高的配置

## 总结

**资源利用率为0%的根本原因：**

1. **数学原因**：边际收益递减（`1 - exp(-E_i)` 是凹函数）
2. **权重原因**：某些资源权重太低，贡献小
3. **竞争原因**：其他资源已提供足够保护，边际收益更小
4. **优化原因**：优化器追求最大化总收益，而不是最大化资源使用率

**这不是bug，而是优化器的正确行为！**

如果需要改变这种行为：
- 调整权重配置
- 增加覆盖半径
- 减少其他资源数量
- 或者接受优化结果（推荐）
